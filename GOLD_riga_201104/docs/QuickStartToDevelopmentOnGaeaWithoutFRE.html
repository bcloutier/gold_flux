GOLD / Quick Start to Development on Gaea Without FRE

Example checkout and build on Gaea

The following assumes you are using certificates to access the cvs repository. Your environment should look like this: 

CVS_RSH=gsissh
CVSROOT=:ext:cvs.princeton.rdhpcs.noaa.gov:/home/fms/cvs

To install and build GOLD, cut and paste the following in to a shell on a gaea login node: 


mkdir -p extras
cvs co -kk -r GOLD_ogrp -P GOLD
cvs co -kk -r riga_201104 -P shared
cvs co -kk -r fre-commands-arkansas-9 -P -d site fre/fre-commands/site
cvs co -kk -r fre-commands-arkansas-9 -P -d bin bin-pub
(cd extras; cvs co -kk -r riga_201104 -P -d GOLD_SIS atmos_null atmos_param/diag_integral atmos_param/monin_obukhov coupler ice_sis ice_param land_null)
(cd extras; cvs up -r riga_201104 -P -d GOLD_SIS/ice_sis)
(cd extras; cvs co -kk -r riga_201104 -P -d CM2G atmos_coupled atmos_fv_dynamics atmos_param atmos_shared coupler ice_param land_lad land_param)
(cd extras/CM2G; ln -s ../GOLD_SIS/ice_sis .)
rm -rf extras/CM2G/atmos_fv_dynamics/driver/solo

module unload PrgEnv-pgi
module unload PrgEnv-intel
module load PrgEnv-pathscale
module load netcdf/4.0.1.3
set compiler = pathscale

mkdir -p build
(cd build; mkdir -p shared solo GOLD_SIS CM2G debug.solo debug.GOLD_SIS  debug.CM2G solo.symmetric)

cd build/shared
rm path_names
../../bin/list_paths ../../shared/
mv path_names pt_orig
egrep -v "atmos_ocean_fluxes|coupler_types|coupler_util" pt_orig > path_names
../../bin/mkmf -t ../../site/ncrc/$compiler.mk -p libfms.a -c "-Duse_libMPI -Duse_netCDF" path_names
make NETCDF=4 REPRO=1 -j 6

cd ../solo
ln -sf ../shared/*.{o,mod} .
rm path_names
../../bin/list_paths ../../GOLD/config_src/dynamic/ ../../GOLD/config_src/solo_driver/ ../../GOLD/src/ ../../shared/
../../bin/mkmf -t ../../site/ncrc/$compiler.mk -p GOLD -c "-Duse_libMPI -Duse_netCDF" path_names
make NETCDF=4 REPRO=1 -j 6

cd ../GOLD_SIS
ln -sf ../shared/*.{o,mod} .
rm path_names
../../bin/list_paths ../../GOLD/config_src/dynamic/ ../../GOLD/config_src/coupled_driver/ ../../GOLD/{examples/GOLD_SIS/,src/} ../../shared/ ../../extras/GOLD_SIS/
../../bin/mkmf -t ../../site/ncrc/$compiler.mk -p GOLD -c "-Duse_libMPI -Duse_netCDF" path_names
make NETCDF=4 REPRO=1 -j 6

cd ../CM2G
ln -sf ../shared/*.{o,mod} .
rm path_names
../../bin/list_paths ../../GOLD/config_src/dynamic/ ../../GOLD/config_src/coupled_driver/ ../../GOLD/src/ ../../shared/ ../../extras/CM2G/*/
../../bin/mkmf -t ../../site/ncrc/$compiler.mk -p GOLD -c "-Duse_libMPI -Duse_netCDF -DSPMD -DLAND_BND_TRACERS" path_names
make NETCDF=4 REPRO=1 -j 6

cd ../..

To run any of the configurations, you need to login access the batch nodes either interactively 

msub -I -l size=24 -l walltime=02:00:00

or via a batch job script. Note that size=24 is the minimum and cores come in multiples of 24.

To run double_gyre: 

cd GOLD/examples/double_gyre
aprun -n 8 ../../../build/solo/GOLD

To run DOME: 

cd GOLD/examples/DOME
aprun -n 6 ../../../build/solo/GOLD

To run global: 

cd GOLD/examples/global
ln -sf /archive/gold/datasets/global/omsk/INPUT .
aprun -n 36 ../../../build/solo/GOLD

To run GOLD_SIS: 

cd GOLD/examples/GOLD_SIS
mkdir -p INPUT
(cd INPUT; ln -sf /archive/gold/datasets/GOLD_SIS/omsk_2007_10/INPUT/* .)
mkdir -p RESTART; rm -f *.nc* RESTART/*
aprun -n 30 ../../../build/GOLD_SIS/GOLD

To run CM2G: 

cd GOLD/examples/CM2G
mkdir INPUT RESTART
(cd INPUT; ln -sf /archive/gold/datasets/CM2G/perth/INPUT/* .)
(cd INPUT; ln -sf /archive/gold/datasets/CM2G/perth/mosaic.mica.unpacked/* .)
(cd INPUT; ln -sf /archive/gold/datasets/CM2G/perth/RESTART/CM2G.initCond_2008.06.04.unpacked/* .)
aprun -n 120 ../../../build/CM2G/GOLD

To run CM2G63L: 

cd GOLD/examples/CM2G63L
mkdir INPUT RESTART
(cd INPUT; ln -sf /archive/gold/datasets/CM2G/perth/INPUT/* .)
(cd INPUT; ln -sf /archive/gold/datasets/CM2G/perth/mosaic.mica2.unpacked/* .)
(cd INPUT; ln -sf /archive/gold/datasets/CM2G/perth/RESTART/CM2G.initCond_2008.06.04.unpacked/* .)
aprun -n 120 ../../../build/CM2G/GOLD
